### Round 2 — full-project statistical audit (2026-07-12)

Findings from a full statistical-correctness audit of the package (four parallel code
reviews over the power layer, optimality engine, analysis layer, and design-structure
modules, plus independent Monte Carlo validation of the core power claims). All P1
findings were independently reproduced before ticketing; every finding below is
numerically CONFIRMED unless marked otherwise.

**Verified correct (do not re-audit without cause):** core power formulas match Monte
Carlo simulation — OLS joint contrast, global R² (λ = f²·n), split-plot GLS joint
contrast vs classical within-plot ANOVA, GLM logistic Wald (6k–200k reps each);
I/D/A criterion values and all Fedorov rank-one updates match brute force to machine
precision; split-plot stratum df reproduce classical balanced ANOVA df; Woodbury
inverse exact (balanced and unbalanced); WP-nesting invariant cannot be violated by the
exchange; n-search rebuilds the design at every n (λ and df consistent) and returned n
verified minimal by brute force; `min_detectable_effect` inverts the same power function
it reports against (power at MDE = 0.80002 vs 0.80); `contrast_from_scenarios` exact
(apart from TD-7); `glm_fisher_weight` correct for binomial/Poisson; Hotelling T² exact
for s = 1 and equal to `contrast_power` at k = 1; VIF/leverage/D-efficiency standard.

## Ideas / Future Consideration

Add rough ideas here before they are fleshed out enough to promote to the backlog.

- Simulation-based (Monte Carlo) power for non-normal residuals
- Bayesian D-optimal (incorporate prior beliefs on coefficients)
- Design catalog — pre-computed designs for common factor structures (e.g., 2³ full factorial, Box-Behnken, CCD)
- Export designs to JMP / Minitab / SAS format
- GPU acceleration for parallel starts (large grid problems)
- Design sensitivity to model misspecification (e.g., wrong functional form)
- **Point-wise GLM-optimal design (SR-2):** Replace the scalar Fisher weight `w` (evaluated once at the null baseline) with per-point weights `wᵢ = p(xᵢ, β̂)(1−p(xᵢ, β̂))` derived from a nominal parameter vector. This is the standard locally D-optimal GLM design criterion and gives a more accurate information matrix for designs with substantial slopes and wide covariate ranges. Requires extending the Fedorov exchange to accept a weight vector and a nominal parameter input.
- **Satterthwaite / Kenward-Roger denominator df for split-plot power (SR-3):** Replace the WP-vs-SP stratum classification heuristic in `contrast_power_sp` with a Satterthwaite or Kenward-Roger df calculation. This is important for unbalanced split-plot designs and configurations with multiple variance components. Would require estimating the Satterthwaite df from the GLS information matrix and the variance component structure.
- **Correlated multi-response power beyond OLS shared-formula (SR-4):** Extend `sigma_joint` / Hotelling T² support to cover (a) GLM multi-response runs and (b) multi-response split-plot designs. Currently both combinations are explicitly rejected. A principled extension would require a multivariate Wald test that incorporates the GLS variance structure for split-plot or the GLM Wald chi-square covariance for GLM responses.
- **Narrow `__init__` export surface (AR-4):** The top-level package re-exports low-level internals (`build_whole_plot_indicator`, `build_split_plot_covariance_inv`, `gls_information_matrix`, `build_candidate`, `build_split_plot_candidate`, `build_model_matrix`) that most users will never call directly. Narrowing `__all__` to the core analytical surface and moving optional integrations (Sheets, Excel, widgets, split-plot covariance utilities) behind explicit submodule imports would give the public API a cleaner shape. Not urgent; would require a deprecation cycle if any downstream callers rely on top-level access. Key file: `lattice_doe/__init__.py`.
